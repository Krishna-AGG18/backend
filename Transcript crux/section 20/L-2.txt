SQL (Structured Query Language):
Uses a fixed schema
Data stored in tables (rows & columns) like Excel
Tables are connected using primary keys & foreign keys
Also called relational databases (RDBMS)
Best for structured, predictable data
Examples: MySQL, PostgreSQL

NoSQL (Not Only SQL):
Uses flexible schema
Data stored as documents (JSON-like objects)
Different records can have different fields
Easy to evolve data structure over time
Best for dynamic, rapidly changing data
Example: MongoDB

Key Difference:
SQL ‚Üí strict structure, strong relationships
NoSQL ‚Üí flexible structure, faster iteration
