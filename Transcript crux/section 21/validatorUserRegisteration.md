# 🔹 What is Validation?

Validation means:

> “Checking whether incoming data is correct or not before saving/processing it.”

Example:

```js
email: "abc@gmail.com" ✅
email: "abcgmail" ❌
password: "123" ❌
password: "Strong@123" ✅
```

Even though all are strings, not all are **valid** strings.

---

# 🔹 Why Validation is Important?

Without validation:

* bad data enters DB
* wrong emails get saved
* empty passwords accepted
* security issues happen
* app crashes later

So validation acts like a **security guard** before controller logic runs.

---

# 🔹 Libraries for Validation

The video mentioned:

* Zod
* Yup
* Express Validator ✅ (used here)

They all do same work:

✔ Validate data
✔ Return errors
✔ Stop invalid requests

Only syntax differs.

# 🔹 What is Middleware?

Middleware = function that runs **between request and response**

Flow:

```txt
Client Request
      ↓
Middleware
      ↓
Controller
      ↓
Response
```

Example:

```txt
Request → Validation Middleware → registerUser Controller
```

Middleware can:

* validate data
* check JWT token
* check admin role
* log requests
* sanitize input


### isLength()

### optional()

```js

  .notEmpty()
```

# 🔹 Difference Between Validator & Middleware

Many beginners confuse this.

---

## Validator

```js
userRegisterValidator()
```

ONLY creates validation rules.

It says:

```txt
email should be valid
password required
username lowercase
```

It DOES NOT handle errors.

---

## Middleware

```js
validate
```

This actually:

* checks collected errors
* formats them
* throws error
* or calls next()

---

# 🔹 Why validate is NOT called like validate()

Because Express automatically passes:

```js
(req, res, next)
```

to middleware.

So:

```js
validate
```

is enough.

---

# 🔹 Why userRegisterValidator() IS called?

Because it RETURNS an array of validators.

Without `()`:

```js
userRegisterValidator
```

would just reference the function.

With `()`:

```js
userRegisterValidator()
```

it executes and returns validator chains.
