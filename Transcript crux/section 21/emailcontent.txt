📧 Email Utility Explained (from your lecture transcript)

1. Why Email Utilities Are Needed
   When a user registers, the system usually follows these steps:

Step 1 – Take user data
Example: username, email, password.

Step 2 – Validate the data
Check if fields are correct (email format, password length, etc.).

Step 3 – Check the database
Verify if a user with that email already exists.

Step 4 – Save the new user
If validation passes and the user doesn’t exist, store the user in the database.

Step 5 – Verify the user
After saving the user, the application sends an email to verify their account.

This verification process requires sending an email containing a verification link.
That link includes the temporary token we generated earlier.

Example flow:
User registers → server generates token → server sends email → user clicks link → account verified.

---

2. Why We Use Mailgen
   To send emails, we first need to **generate email content**.
   For this, the course uses the **Mailgen** library.

Mailgen helps create **beautiful formatted emails** automatically.

Install it:

npm install mailgen

Mailgen does NOT send emails.
It only generates the **email template and content**.

Actual email sending will be done later using another service.

---

3. Creating the Email Utility File

Inside the `utils` folder create:

mail.js

Import Mailgen:

import Mailgen from "mailgen";

This file will contain functions that generate email content.

---

4. Email Verification Content Generator

Function:

emailVerificationMailgenContent(username, verificationUrl)

Purpose:
Generate the email body for verifying a new user's email.

It returns an object describing the email structure.

Structure example:

return {
body: {
name: username,

```
intro: "Welcome to our app. We are excited to have you on board.",

action: {
  instructions: "To verify your email please click the button below",

  button: {
    color: "#22BC66",
    text: "Verify your email",
    link: verificationUrl
  }
},

outro: "Need help? Just reply to this email and we would love to help."
```

}
}

Explanation:

name → user's name shown in the email
intro → greeting message
action → contains the button and instructions
button → clickable verification button
link → URL where the button will redirect
outro → closing message

---

5. Forgot Password Email Content Generator

Another function is created for password reset.

Function:

forgotPasswordMailgenContent(username, passwordResetUrl)

Structure:

return {
body: {
name: username,

```
intro: "We received a request to reset your password.",

action: {
  instructions: "To reset your password click the button below",

  button: {
    color: "#FF6136",
    text: "Reset Password",
    link: passwordResetUrl
  }
},

outro: "If you did not request this password reset, you can safely ignore this email."
```

}
}

Difference from verification email:
• Different message
• Different button text
• Different link (password reset link)

---

6. Exporting These Functions

At the end of the file:

export {
emailVerificationMailgenContent,
forgotPasswordMailgenContent
}

Now these functions can be imported anywhere in the project.

Example:

import {
emailVerificationMailgenContent,
forgotPasswordMailgenContent
} from "../utils/mail.js";

---

7. Important Concept

This file **does NOT send emails**.

It only prepares email content.

Actual email sending will be implemented later using a mail service.

So the workflow will be:

Controller → generate token → generate email content → send email → user receives link.

---

8. Summary

This step creates reusable utilities for email templates.

Two email templates are generated:
• Email verification
• Password reset

Mailgen helps build professional email layouts with:
• greeting text
• instructions
• action button
• closing message

In the next step, the backend will actually **send these generated emails to users.**
